# Gap Report: SAGT Evolution to v3.0

## 1. Current State (v2.2)
- Monolithic `docs/` folder.
- Basic YAML rule compilation.
- HITL markers present.
- Spec Mode via `.jtasks/`.

## 2. Desired State (v3.0)
- Functional Multi-Zone Governance (`governance/`, `canon/`, `policy/`, `reference/`, `jules/`, `compiler/`).
- Formal Deep Governance Mode (Validation Loop).
- Schema-driven State Machine.
- Integrated Conformance Suite (Python + Shell).

## 3. Distance
- The distance is **Significant**. It requires a full reorganization of the governance filesystem and the introduction of automated validation tools that move beyond simple grep checks.
