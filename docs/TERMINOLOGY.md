# Project Name | Terminology (Canonical)

## 0) Core Concepts
- **Canon**: The authoritative set of rules and models (stored in `docs/`).
- **Plane**: A conceptual layer of the system with its own authority and boundaries.
- **Invariants**: Truths that must remain unchanged regardless of system state.
- **Contract**: An immutable interface between two planes or components.
- **Refusal**: The mandatory agent behavior when a request violates canon.
- **Skill**: A deterministic procedure for performing a specific task.
- **Spec**: A formal plan for a unit of work (stored in `.jtasks/`).
- **Task**: An atomic unit of execution within a spec.
